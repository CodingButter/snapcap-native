# Interface: IFriendsManager

Defined in: [api/friends.ts:235](https://github.com/CodingButter/snapcap-native/blob/3189675f17f92b8303422ec60867fb61b844da17/src/api/friends.ts#L235)

Friends domain manager — all friend-graph operations live here.

All UUIDs are hyphenated string [UserId](../type-aliases/UserId.mdx) values. Mutations
resolve `void` on success; reads return consumer-shape types
([Friend](Friend.mdx), [User](FriendsUser.mdx), [FriendRequest](FriendRequest.mdx),
[OutgoingRequest](OutgoingRequest.mdx)) — never bundle protobuf shapes.

## Remarks

Reads share a single underlying [IFriendsManager.snapshot](#snapshot).
[IFriendsManager.list](#list), [IFriendsManager.incomingRequests](#incomingrequests),
and [IFriendsManager.outgoingRequests](#outgoingrequests) are slim accessors that
project the relevant slice. One subscription method
([IFriendsManager.onChange](#onchange)) fires whenever any of the three
slots changes; it returns an `Unsubscribe` thunk that's
idempotent.

Pending-request methods ([IFriendsManager.acceptRequest](#acceptrequest),
[IFriendsManager.rejectRequest](#rejectrequest)) are gated on the chat-bundle
source-patch surfacing the closure-private `FriendRequests` client
(planned global: `__SNAPCAP_FRIEND_REQUESTS_CLIENT`). Until that
lands, those methods throw an explicit "not yet wired" error.

### onChange()

```ts
onChange(cb: (snap: FriendsSnapshot) => void): Unsubscribe;
```

Defined in: [api/friends.ts:410](https://github.com/CodingButter/snapcap-native/blob/3189675f17f92b8303422ec60867fb61b844da17/src/api/friends.ts#L410)

Fire `cb` whenever any part of the friend graph changes — mutuals,
incoming requests, or outgoing requests.

The callback receives a full [FriendsSnapshot](FriendsSnapshot.mdx) reflecting the
new state. Initial state is NOT replayed — call
[IFriendsManager.snapshot](#snapshot) once after subscribing if you need
a baseline.

#### Parameters

| Parameter | Type | Description |
| ------ | ------ | ------ |
| `cb` | (`snap`: [`FriendsSnapshot`](FriendsSnapshot.mdx)) => `void` | Subscriber invoked with the latest snapshot on every relevant change. |

#### Returns

[`Unsubscribe`](../type-aliases/Unsubscribe.mdx)

An `Unsubscribe` thunk; idempotent on repeat calls.

#### Example

```ts
const unsub = client.friends.onChange((snap) => {
  console.log(`mutuals=${snap.mutuals.length}`);
});
// ...later
unsub();
```

Remove a friend from the social graph.

#### Parameters

| Parameter | Type | Description |
| ------ | ------ | ------ |
| `userId` | `string` | Hyphenated UUID of the friend to remove. |

#### Returns

[`Promise`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Promise)\<`void`\>

#### Remarks

Snap unfriend is asymmetric — the recipient unfriending the sender
doesn't sever the link from the sender's side. Subsequent
[IFriendsManager.add](#add) calls against an account that still sees
the caller as mutual become silent no-ops.

#### Example

```ts
await client.friends.remove(userId);
```
