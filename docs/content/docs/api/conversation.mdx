A live chat handle. Wraps a conversation ID + participant set so call sites can use ergonomic methods (`chat.sendText("hi")`, `chat.setTyping(2000)`) without re-passing IDs.

Constructed by `client.getConversations()` (preferred) or `client.conversation(id, [participantIds])` (when you've persisted an ID and want to skip the `SyncConversations` round-trip).

```ts
const conversations = await client.getConversations();
const chat = conversations[0];

await chat.setTyping(2000);
await chat.sendText("hello");
```

## Fields

```ts
public readonly conversationId: string;
public readonly participants: User[];
public readonly kind: ConversationKind;
public readonly lastActivityAt: Date | undefined;
```

```ts
type ConversationKind = "dm" | "group" | "myStory" | "unknown";
```

`participants` includes the logged-in user when the data came from `SyncConversations`. `lastActivityAt` is a `Date` parsed from Snap's millisecond timestamp; `undefined` when constructed via `client.conversation(...)` with only an ID.

## friend (getter)

```ts
get friend: User | undefined
```

The other participant in a DM. `undefined` for groups, my-story, or before the owning client has resolved its self-user.

## Sending

### sendText

```ts
sendText(message: string): Promise<void>
```

Send a text message. Requires the owning client's `self` to be set.

### sendImage

```ts
sendImage(bytes: Uint8Array, opts?: { width?: number; height?: number }): Promise<void>
```

Send an image. The pipeline:

1. `getUploadLocations` → signed S3 PUT URL + media id
2. AES-256-CBC encrypt with a fresh per-message key
3. PUT ciphertext to S3
4. `CreateContentMessage` with the media id + key + IV embedded

PNG / JPEG / WebP dimensions are auto-detected. Pass `width`/`height` for other formats.

### sendImageWithCaption

```ts
sendImageWithCaption(
  bytes: Uint8Array,
  caption: string,
  opts?: { width?: number; height?: number },
): Promise<void>
```

Send an image followed by a text caption. On the wire this is two separate `CreateContentMessage` calls; Snap's UI renders them as one logical "image with caption."

### typeAndSendText

```ts
typeAndSendText(durationMs: number, message: string): Promise<void>
```

Show the typing indicator for `durationMs`, then send the message. Mimics a human composing in the UI.

## Presence

### setTyping

```ts
setTyping(durationMs?: number): Promise<void>
```

Show the "typing…" indicator to the recipient.

- **No argument** — single pulse. Snap's UI auto-clears in ~5 s.
- **`durationMs`** — refreshes every 2.5 s for the duration so the indicator stays on continuously.

Routes over the duplex WebSocket. The gRPC equivalent is accepted server-side but doesn't fan out — `setTyping` is the only thing that actually shows up to recipients.

### markViewed

```ts
markViewed(state?: number): Promise<void>
```

Mark the conversation as actively viewed — recipient sees the "in chat" bitmoji indicator. Routes over the duplex WebSocket.

### markMessageViewed

```ts
markMessageViewed(messageId: bigint, action?: number): Promise<void>
```

Mark a single received message as viewed. `action` defaults to 15 (mark-viewed in captured traffic).

## Constants

```ts
export const TypingActivity = {
  TEXT: 6,
} as const;

export const ConversationViewState = {
  ENTERED: 9,    // entering / focusing the conversation
  ACTIVE:  10,   // actively viewing
} as const;
```

Other values exist on the wire (voice recording, etc.) but aren't surfaced yet — we add them as recon validates them.
