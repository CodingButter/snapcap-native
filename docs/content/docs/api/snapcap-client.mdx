The main entry point. Constructed once per account. Holds the cookie jar, bearer, lazy bundle-native `MessagingSession`, and the resolved `self` user. Every call you make on the SDK starts here.

```ts
import { SnapcapClient, FileDataStore } from "@snapcap/native";

const client = new SnapcapClient({
  dataStore: new FileDataStore("./auth.json"),
  username: process.env.SNAP_USER,
  password: process.env.SNAP_PASS,
});

if (await client.isReady()) {
  const friends = await client.listFriends();
}
```

## Constructor

### `new SnapcapClient(opts)`

**Signature**

```ts
new SnapcapClient(opts: SnapcapClientOpts)

type SnapcapClientOpts = {
  dataStore: DataStore;
  username?: string;
  password?: string;
  userAgent?: string;
};
```

**Parameters**

| Name | Type | Description |
|---|---|---|
| `opts.dataStore` | `DataStore` | Persistent backing for cookies, bearer, and the bundle's own sandbox storage. Required. |
| `opts.username` | `string?` | Cold-start username. Only consulted when no restored cookies are present. **Not** persisted to the DataStore. |
| `opts.password` | `string?` | Cold-start password. Same lifecycle as `username` — pass on every boot if you want auto-recovery from session expiry. |
| `opts.userAgent` | `string?` | UA fingerprint used at login and on subsequent gRPC calls. Defaults to a Linux Chrome 147 string. |

**Behavior**

The constructor is synchronous. It calls `installShims({ url, dataStore })` so the singleton sandbox is seeded with the consumer's DataStore eagerly (otherwise the kameleon boot would race and write to happy-dom's in-memory defaults). No network and no disk I/O beyond the DataStore's own load.

All real work happens inside `isReady()`.

## Public fields

### `client.self`

```ts
public self?: User;
```

The logged-in user. Auto-populated by `isReady()` when it runs a fresh login (via `resolveSelf` walking SyncFriendData) or restored from `local_snapcap_self` on a warm start. Set manually with `setSelf(user)` if you already know who you're logged in as.

## Auth

### `client.isReady(opts?)`

Resolve whether the client has a usable session. Idempotent and cached — the first positive result is held in-memory so subsequent calls are free.

**Signature**

```ts
isReady(opts?: { force?: boolean }): Promise<boolean>
```

**Parameters**

| Name | Type | Description |
|---|---|---|
| `opts.force` | `boolean?` | Bypass the in-memory cache and re-run the login flow even when warm. |

**Returns** — `Promise<boolean>`. Decision flow:

- **Restored cookies + bearer present in the DataStore** → `true` (no network).
- **No cookies but credentials supplied** → run full login, persist, return `true`.
- **No cookies, no creds** → `false`.
- **Server rejects creds** → `false` (does **not** throw).

**Example**

```ts
if (await client.isReady()) {
  // safe to call any API method
} else {
  // creds were missing or rejected — re-prompt or surface to user
}

// Force a fresh login (e.g. after password rotation):
await client.isReady({ force: true });
```

See [auth model](/docs/guide/auth) for the full decision tree.

### `client.logout()`

Clear the SDK's auth-state keys from the DataStore.

**Signature**

```ts
logout(): Promise<void>
```

**Behavior** — Deletes `cookie_jar`, `session_snapcap_bearer` (via `sessionStorage.removeItem("snapcap_bearer")`), and `local_snapcap_self` (via `localStorage.removeItem("snapcap_self")`). Bundle-owned `local_*` / `session_*` / `indexdb_*` keys are deliberately left intact — including the bundle-minted Fidelius wrapped identity at `local_uds_uds.e2eeIdentityKey.shared`. Wiping them would force the next `isReady()` to re-bootstrap WASM state from scratch, which costs an extra round-trip and ~250 ms WASM init.

### `client.setSelf(user)`

Override `client.self` directly.

**Signature**

```ts
setSelf(user: User): void
```

**Use when** — Auto-discovery covers the typical case (a fresh login walks SyncFriendData and finds the logged-in user's record). Reach for this only if you have an out-of-band record of who you're logged in as, or in tests.

### `client.resolveSelf(username)`

Resolve the self-user from server-side data and set `client.self`.

**Signature**

```ts
resolveSelf(username: string): Promise<User>
```

**Parameters**

| Name | Type | Description |
|---|---|---|
| `username` | `string` | The mutable username of the logged-in account. The SyncFriendData response embeds the caller's own record alongside the friend list; this matches on `mutableUsername`. |

**Returns** — `Promise<User>`. Sets `client.self` and returns the same value. **Throws** if the username isn't found in the response.

## Conversations

### `client.getConversations()`

Fetch the list of conversations the logged-in user is in.

**Signature**

```ts
getConversations(): Promise<Conversation[]>
```

**Returns** — `Promise<Conversation[]>`. Each `Conversation` is bound to this client; call methods on it directly without re-passing the client.

**Throws** — if `client.self` isn't set. Call `client.isReady()` first (which auto-resolves self) or `client.setSelf(...)` manually.

**Behavior** — Routes through the bundle's `getFeedManager()` (lazy-brings up the messaging session if needed). Returns the bundle's authoritative feed snapshot, mapped to `Conversation` instances.

**Example**

```ts
await client.isReady();
const chats = await client.getConversations();
for (const chat of chats) {
  console.log(chat.kind, chat.conversationId, chat.participants.length);
}
```

### `client.conversation(conversationId, participantUserIds?)`

Build a `Conversation` handle for a known conversation ID without round-tripping the feed.

**Signature**

```ts
conversation(conversationId: string, participantUserIds?: string[]): Conversation
```

**Parameters**

| Name | Type | Description |
|---|---|---|
| `conversationId` | `string` | Hyphenated UUID of the conversation. |
| `participantUserIds` | `string[]?` | Hyphenated UUIDs of all participants (including self). Required if you want `Conversation.setTyping` / `markViewed` on the legacy duplex path; the bundle path resolves participants itself. Defaults to `[]`. |

**Returns** — `Conversation`. `kind` is set to `"dm"` if `participantUserIds.length === 2`, otherwise `"unknown"`.

**Use when** — You've persisted a conversation ID in your own DB and want to skip the SyncConversations round-trip.

```ts
const chat = client.conversation(persistedId, [client.self!.userId, peerUserId]);
await chat.sendText("hi");
```

## Friends

### `client.listFriends()`

Fetch the logged-in user's friend list.

**Signature**

```ts
listFriends(): Promise<User[]>
```

**Returns** — `Promise<User[]>`. Each `User` includes username, display name, friend-link state, bitmoji metadata, and the raw record under `.raw`. Routes through the bundle's AtlasGw client so the wire bytes match the SPA.

### `client.searchUsers(query, _pageSize?)`

Search Snap's user index by query string.

**Signature**

```ts
searchUsers(query: string, _pageSize?: number): Promise<User[]>
```

**Parameters**

| Name | Type | Description |
|---|---|---|
| `query` | `string` | Search query — username, display name, or partial match. |
| `_pageSize` | `number?` | Ignored. Retained for source-compat. The bundle pins `pageSize` at 20 (the value the SPA sends for every search); the SDK no longer surfaces a knob. |

**Returns** — `Promise<User[]>`. Routes through the bundle's `default-authed-fetch` so requests share Snap's own auth interceptors.

### `client.addFriend(userId, source?)`

Send a friend request.

**Signature**

```ts
addFriend(userId: string | string[], source?: number | string): Promise<void>
```

**Parameters**

| Name | Type | Description |
|---|---|---|
| `userId` | `string \| string[]` | One UUID, or an array — the SDK fans out sequentially. |
| `source` | `number \| string?` | `FriendSource` enum value. Defaults to `FriendSource.ADDED_BY_USERNAME` (2), what the SPA sends from a search-result "Add" button. A string is silently coerced to the default for back-compat. |

**Returns** — `Promise<void>`. Routes through the bundle's `jz.AddFriends` thunk — same wire bytes the SPA emits when a user clicks "Add" on a search result.

**Example**

```ts
import { FriendSource } from "@snapcap/native";

const [target] = await client.searchUsers("teamsnapchat");
await client.addFriend(target.userId, FriendSource.ADDED_BY_USERNAME);
```

## Messages

### `client.sendText(conversationId, text)`

Send a text DM into a conversation.

**Signature**

```ts
sendText(conversationId: string, text: string): Promise<void>
```

**Parameters**

| Name | Type | Description |
|---|---|---|
| `conversationId` | `string` | Hyphenated UUID of the conversation. |
| `text` | `string` | Message body. UTF-8. The SDK enforces no length cap; Snap's server has its own. |

**Returns** — `Promise<void>`. Resolves when the bundle's send pipeline acknowledges the message.

**Behavior** — Routes through the bundle's natural `sendText` entry (module 56639 export `pn`) so the wire message is byte-identical to one sent by the official web client.

```ts
await client.isReady();
const [chat] = await client.getConversations();
await client.sendText(chat.conversationId, "hello world");
```

### `client.sendImage(conversationId, bytes)`

Send a DM image.

**Signature**

```ts
sendImage(conversationId: string, bytes: Uint8Array): Promise<void>
```

**Parameters**

| Name | Type | Description |
|---|---|---|
| `conversationId` | `string` | Hyphenated UUID of the conversation. |
| `bytes` | `Uint8Array` | Raw image bytes. PNG / JPEG / WebP all accepted. |

**Returns** — `Promise<void>`. Routes through the bundle's `sendImage` entry (module 56639 export `E$`) — encryption, upload, dimension extraction, and `CreateContentMessage` are all owned by the bundle.

### `client.postStory(bytes, _opts?)`

Post an image to MY_STORY (the user's own story feed).

**Signature**

```ts
postStory(bytes: Uint8Array, _opts?: { skipNormalize?: boolean }): Promise<void>
```

**Parameters**

| Name | Type | Description |
|---|---|---|
| `bytes` | `Uint8Array` | Raw image bytes. |
| `_opts.skipNormalize` | `boolean?` | Ignored. Preserved for source-compat — the bundle's media pipeline handles dimension normalisation now. Custom-sized bytes still work. |

**Returns** — `Promise<void>`. Routes through the bundle's `sendSnap` entry with a MY_STORY destination.

### `client.fetchMessages(conversationId)`

Fetch recent messages for a conversation.

**Signature**

```ts
fetchMessages(conversationId: string): Promise<FetchMessagesViaSessionResponse>

type FetchMessagesViaSessionResponse = {
  messages: unknown;            // bundle-realm Map<MessageId, MessageRecord>
  conversation: unknown;        // bundle-realm Conversation record
  hasMoreMessages: boolean;     // true when older pages exist
};
```

**Parameters**

| Name | Type | Description |
|---|---|---|
| `conversationId` | `string` | Hyphenated UUID of the conversation. |

**Returns** — The bundle's natural `{messages, conversation, hasMoreMessages}` shape. `messages` is the bundle's Map-style collection of message records — iterate it and run each `.content` through `session.codec.decode` (or use [`onMessage`](#client-onmessage-handler) for the live stream). `hasMoreMessages` is true when older pages can be fetched (paginated sibling not surfaced yet).

**Behavior** — Routes through `fetchConversationWithMessages` (module 56639 export `uk`) so the WASM session's local cache is consulted first and the server fetch shares Snap's delegate dispatch + MessageContent codec.

> The legacy `limit` / `secondary` opts are no longer surfaced — the bundle wrapper has no equivalent knobs.

### `client.onMessage(handler)`

Subscribe to inbound messages.

**Signature**

```ts
onMessage(handler: (msg: DecodedMessage) => void): Promise<Unsubscribe>

type Unsubscribe = () => void;
```

**Parameters**

| Name | Type | Description |
|---|---|---|
| `handler` | `(msg: DecodedMessage) => void` | Called once per envelope the bundle's delegate harness pumps out — text, image, sticker, system events, etc. See [DecodedMessage](/docs/api/messages#decodedmessage). |

**Returns** — `Promise<Unsubscribe>`. Call the returned function to detach the listener. Safe to register multiple handlers; each fires independently.

**Behavior** — The pump is the same EventEmitter the bundle's slot-6 (`analyticsLogger`) and slot-9 (`messagingDelegate`) feed. There's no polling, no extra RPC. Handler exceptions are swallowed (one bad consumer shouldn't kill the pump for the rest of the session). `decoded` is `null` if the codec couldn't decode the envelope's `.content` (receipts and other metadata-only envelopes pass through with `decoded: null`).

```ts
const unsubscribe = await client.onMessage((msg) => {
  if (msg.isSender) return;                        // outbound echo — skip
  const text = msg.decoded?.content?.text?.text;
  if (text) console.log(`<${msg.conversationId}> ${text}`);
});

// later:
unsubscribe();
```

## Presence

### `client.markMessageViewed(messageId, conversationId, _userId?, action?)`

Mark a single received message as viewed.

**Signature**

```ts
markMessageViewed(
  messageId: string | { id: Uint8Array },
  conversationId: string,
  _userId?: string,
  action?: number,
): Promise<void>
```

**Parameters**

| Name | Type | Description |
|---|---|---|
| `messageId` | `string \| { id: Uint8Array }` | Hyphenated UUID string (the bundle's `attemptId.id` shape — what `onMessage` surfaces as `msg.messageId`) or a `{ id: bytes16 }` envelope. |
| `conversationId` | `string` | Hyphenated UUID of the conversation. |
| `_userId` | `string?` | Ignored — the bundle pulls it from the session itself. Retained for source-compat. |
| `action` | `number?` | Display state. Defaults to `1` (`MESSAGE_DISPLAY_STATE_DISPLAYED` — the value the SPA sends for "user saw it"). |

**Returns** — `Promise<void>`. **Throws** if `messageId` is a `bigint` (the legacy raw-gRPC shape is no longer accepted on the bundle path).

**Behavior** — Routes through `mediaMessagesDisplayed` (module 56639 export `Sd`) — same callback the SPA fires when the user opens a chat with unread media.

### `client.sendTypingNotification(conversationId, _userId?, _typingType?)`

Show the "typing…" indicator to the recipient.

**Signature**

```ts
sendTypingNotification(
  conversationId: string,
  _userId?: string,
  _typingType?: number,
): Promise<void>
```

**Parameters** — Both `_userId` and `_typingType` are ignored (bundle resolves them). Pass any value for source-compat.

**Returns** — `Promise<void>`. Single pulse; Snap's UI auto-clears in ~3 s. For a continuous indicator, prefer [`Conversation.setTyping(durationMs)`](/docs/api/conversation#setTyping).

**Behavior** — Routes through the bundle's `sendTypingNotification` (module 56639 export `zM`); the bundle owns the duplex WS + Comlink dispatch.

### `client.updateConversationView(conversationId, _userId?, state?)`

Enter / exit a conversation — recipient sees the "in chat" indicator update.

**Signature**

```ts
updateConversationView(
  conversationId: string,
  _userId?: string,
  state?: number,
): Promise<void>
```

**Parameters**

| Name | Type | Description |
|---|---|---|
| `conversationId` | `string` | Hyphenated UUID. |
| `_userId` | `string?` | Ignored. |
| `state` | `number?` | Defaults to `ConversationViewState.ACTIVE` (10). Pass `0` to fire `exitConversation` instead. |

**Returns** — `Promise<void>`. Routes through `enterConversation` / `exitConversation` on the bundle's ConversationManager.

## Low-level RPC

For advanced consumers calling Snap services not yet wrapped in a high-level method.

### `client.rpc` (getter)

```ts
get rpc: { unary(method: GrpcMethodDesc, request: unknown): Promise<unknown> }
```

A lazily-built `rpc.unary` impl bound to this client's jar / bearer / refresh. Shorthand for `client.makeRpc()` with no header transform.

### `client.makeRpc(transformHeaders?)`

Build a fresh `rpc.unary` impl.

**Signature**

```ts
makeRpc(transformHeaders?: HeaderTransform): {
  unary: (method: GrpcMethodDesc<unknown, unknown>, request: unknown) => Promise<unknown>;
};

type HeaderTransform = (headers: Record<string, string>) => Record<string, string>;
```

**Use when** — Some Snap services 401 if `Origin` / `Referer` are present; the helper strips them.

```ts
const stripped = client.makeRpc((h) => {
  delete h["origin"];
  delete h["referer"];
  return h;
});
await someApiCall(stripped, ...);
```

### `FideliusIdentityBlob` (deprecated)

```ts
type FideliusIdentityBlob = {
  publicKey: string;     // hex (65 bytes, 0x04-prefixed P-256)
  privateKey: string;    // hex (32 bytes)
  identityKeyId: string; // hex (32 bytes)

  version: number;

```

Legacy serialized form from the pre-bundle SDK. Kept exported for backwards-compat with consumers that imported it. The SDK no longer reads or writes this blob — the chat bundle owns identity bootstrap. **Do not use in new code; will be removed in a follow-up phase.**
