# Class: Friends

Defined in: [api/friends.ts:598](https://github.com/CodingButter/snapcap-native/blob/3189675f17f92b8303422ec60867fb61b844da17/src/api/friends.ts#L598)

Concrete [IFriendsManager](../interfaces/IFriendsManager.mdx) implementation.

Constructed once per [SnapcapClient](SnapcapClient.mdx) and held as
[SnapcapClient.friends](SnapcapClient.mdx#friends). See [IFriendsManager](../interfaces/IFriendsManager.mdx) for the
full method-level documentation.

## Remarks

Why a `() => Promise<ClientContext>` provider instead of a bare
`ClientContext`: the SDK's context is built asynchronously (cookie
jar load, etc.), but the `client.friends` field needs to exist
synchronously off `new SnapcapClient(...)`. The provider defers
resolution until the first method call — which is when
`authenticate()` has typically already been called and the context
is warm.


Defined in: [api/friends.ts:730](https://github.com/CodingButter/snapcap-native/blob/3189675f17f92b8303422ec60867fb61b844da17/src/api/friends.ts#L730)

Fire `cb` whenever any part of the friend graph changes — mutuals,
incoming requests, or outgoing requests.

The callback receives a full [FriendsSnapshot](../interfaces/FriendsSnapshot.mdx) reflecting the
new state. Initial state is NOT replayed — call
[IFriendsManager.snapshot](../interfaces/IFriendsManager.mdx#snapshot) once after subscribing if you need
a baseline.

#### Parameters

| Parameter | Type | Description |
| ------ | ------ | ------ |
| `cb` | (`snap`: [`FriendsSnapshot`](../interfaces/FriendsSnapshot.mdx)) => `void` | Subscriber invoked with the latest snapshot on every relevant change. |

#### Returns

An `Unsubscribe` thunk; idempotent on repeat calls.

#### Remarks

Snap unfriend is asymmetric — the recipient unfriending the sender
doesn't sever the link from the sender's side. Subsequent
[IFriendsManager.add](../interfaces/IFriendsManager.mdx#add) calls against an account that still sees
the caller as mutual become silent no-ops.
