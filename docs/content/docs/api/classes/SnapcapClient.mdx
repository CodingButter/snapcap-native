# Class: SnapcapClient

Defined in: [client.ts:143](https://github.com/CodingButter/snapcap-native/blob/main/src/client.ts#L143)

Concrete [ISnapcapClient](../interfaces/ISnapcapClient.mdx) implementation ‚Äî main SDK entry point.

Each instance owns its own isolated `vm.Context` sandbox, Zustand
state, bearer token, and webpack runtime cache, so multiple clients
can coexist in one process without leaking state across tenants.

Method-level documentation (auth verbs, manager fields) lives on
[ISnapcapClient](../interfaces/ISnapcapClient.mdx). The constructor and constructor options are
documented here.

### authenticate()

```ts
authenticate(): Promise<void>;
```

Defined in: [client.ts:239](https://github.com/CodingButter/snapcap-native/blob/main/src/client.ts#L239)

Bring up the bundles, run warm-or-cold WebLogin, and populate the
Zustand auth slice.

The first call does the heavy lifting (loads the accounts and chat
bundles, runs kameleon attestation, drives `WebLoginService` if no
cookies are cached); subsequent process boots with restored cookies
short-circuit through the warm SSO path (~1 redirect, ~100ms).

#### Returns

[`Promise`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Promise)\<`void`\>

Resolves when the auth slice reports `LoggedIn`.

### logout()

```ts
logout(force?: boolean): Promise<void>;
```

Defined in: [client.ts:251](https://github.com/CodingButter/snapcap-native/blob/main/src/client.ts#L251)

Tear down the bundle-side auth state.

Calls Snap's own `state.auth.logout` thunk ‚Äî clears Zustand, fires
any subscribed teardown hooks, and (best-effort) revokes server-side.
Also deletes the persisted cookie jar entry from the DataStore.

#### Parameters

| Parameter | Type | Description |
| ------ | ------ | ------ |
| `force?` | `boolean` | If `true`, force the logout even if the server-side revoke call fails. Defaults to `false`. |

#### Returns

[`Promise`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Promise)\<`void`\>

Resolves when local auth state has been cleared.

refreshAuthToken(): Promise<void>;

Defined in: [client.ts:264](https://github.com/CodingButter/snapcap-native/blob/main/src/client.ts#L264)

Refresh the bearer in-place via Snap's own `state.auth.refreshToken`.

Requires the client to have been constructed with credentials ‚Äî the
bundle's refresh path mints a fresh kameleon attestation bound to
the active identifier.
