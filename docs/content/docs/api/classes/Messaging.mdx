### on()

```ts
on<K>(
   event: K, 
   cb: MessagingEvents[K], 
   opts?: {
  signal?: AbortSignal;
}): Subscription;
```

Defined in: [api/messaging.ts:178](https://github.com/CodingButter/snapcap-native/blob/main/src/api/messaging.ts#L178)

Subscribe to a messaging event. First subscription triggers the
bundle session bring-up (~3s cold; subsequent subscriptions are free).

#### Type Parameters

| Type Parameter |
| ------ |
| `K` *extends* keyof [`MessagingEvents`](../type-aliases/MessagingEvents.mdx) |

#### Parameters

| Parameter | Type | Description |
| ------ | ------ | ------ |
| `event` | `K` | One of [MessagingEvents](../type-aliases/MessagingEvents.mdx). |
| `cb` | [`MessagingEvents`](../type-aliases/MessagingEvents.mdx)\[`K`\] | Callback invoked with the event payload. |
| `opts?` | \{ `signal?`: [`AbortSignal`](https://developer.mozilla.org/docs/Web/API/AbortSignal); \} | Optional `signal`; aborting it unsubscribes. |
| `opts.signal?` | [`AbortSignal`](https://developer.mozilla.org/docs/Web/API/AbortSignal) | - |

#### Returns

[`Subscription`](../type-aliases/Subscription.mdx)

#### Example

```ts
client.messaging.on("message", (msg) => {
  const text = new TextDecoder().decode(msg.content);
  console.log(`${msg.isSender ? "->" : "<-"} ${text}`);
});
```

### setTyping()

```ts
setTyping(convId: string, durationMs: number): Promise<void>;
```

Defined in: [api/messaging.ts:208](https://github.com/CodingButter/snapcap-native/blob/main/src/api/messaging.ts#L208)

Show typing indicator in `convId` for `durationMs`. Returns a
Promise resolving when `durationMs` elapses, so consumers can
compose:

```ts
await messaging.setTyping(convId, 1500);
await messaging.send(convId, "hello");
```

#### Parameters

| Parameter | Type |
| ------ | ------ |
| `convId` | `string` |
| `durationMs` | `number` |

#### Returns

[`Promise`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Promise)\<`void`\>

#### Remarks

Stub today — the underlying WS frame isn't yet wired (the bundle's
outbound presence path lives on a sibling manager off the session
object; mapping pending). Resolves after `durationMs` so the
compose-await pattern works without firing a real frame.
